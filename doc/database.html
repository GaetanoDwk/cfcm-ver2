Database

Di seguito sono riportate le tabelle più importanti al funzionamento di CFCM. E' stata impostata una struttura gerarchica a partire dal cliente. Le tabelle sono correlate tra loro tramite i campi di chiavi esterne (ex_id_**)



Altrettanto importanti ma non correlate tra loro, vi sono le rimanenti tabelle relative a funzionalità aggiuntive e/o di supporto al lavoro di un tecnico forense.



Descrizione tabelle

CLIENTE
Campo Descrizione
cli_id: int Id auto-incrementante.
cli_nome: varchar Nome del cliente (Es: Procura della Repubblica Tribunale di Napoli)
cli_citta: varchar Città in cui si trova il cliente
is_ctp: tinyint 0 | 1. Se abilitato a 1 il cliente è una ctp
is_procura: tinyint 0 | 1. Se abilitato a 1 il cliente è una procura
is_tribunale: tinyint 0 | 1. Se abilitato a 1 il cliente è una tribunale




PM
Campo Descrizione
pm_id: int Id auto-incrementante.
pm_nome: varchar Nome del PM (pubblico ministero)
pm_cognome: varchar Cognome del PM
ex_id_cli: tinyint Chiave esterna relativa all'ID del cliente




CASO
Campo Descrizione
ca_id: int Id auto-incrementante
ca_num: varchar Numero del caso / anno
ca_inc: varchar Incarico. (giorno o numero dell'incarico)
ca_tipo: varchar Tipologia del caso (penale | civile)
ca_dss: varchar Etichetta del disco in cui sono conservati i dati del caso corrente
ca_datains: date Data in cui è stato inserito il caso nel DB
ca_datadel: date Data di eliminazione del caso
ca_ggres: smallint Giorni restanti alla scadenza
ex_id_pm: int Chiave esterna relativa all'ID del PM a cui appartiene il caso corrente




INDAGATO
Campo Descrizione
ind_id: int Id auto-incrementante
ind_titolo: varchar Tipologia (indagato, persona offesa, imputato, proprietario dispositivo)
ind_nome: varchar Nome
ind_cognome: varchar Cognome
ex_id_caso: int Chiave esterna relativa all'Id del caso a cui appartiene l'indagato corrente




HOST
Campo Descrizione
ho_id: int Id auto-incrementante
ho_etichetta: varchar Etichetta dell'host (es. PCASS, CELSAM, ecc...)
ho_seriale: varchar Serial number del dispositivo
ho_pwd: varchar Password del dispositivo
ho_pwd_used: tinyint 0 | 1. Se il campo è 1 allora la password è stata utilizzata per le operazioni di acquisizione.
ho_tipo: varchar Tipologia di Host (ES: Workstation, Notebook, ecc...)
ho_modello: varchar Model number del dispositivo
ho_pathfoto: varchar Percorso della directory in cui si trovano le foto relative all'host corrente
ho_image1: varchar Nome della prima immagine (comprensivo di estensione) dell'host corrente
ho_image2: varchar Nome della seconda immagine (comprensivo di estensione) dell'host corrente
ho_image3: varchar Nome della terza immagine (comprensivo di estensione) dell'host corrente
ho_image4: varchar Nome della quarta immagine (comprensivo di estensione) dell'host corrente
ho_image_docx: varchar Nome della prima imagine scelta per la visualizzazione delle info utili ad essere incollate in consulenza (documento .docx)
ho_image_docx2: varchar Nome della seconda imagine scelta per la visualizzazione delle info utili ad essere incollate in consulenza (documento .docx)
ex_id_caso: int Chiave esterna relativa all'Id del caso a cui appartiene l'host corrente
ex_id_indagato: int Chiave esterna relativa all'Id dell' indagato a cui appartiene l'host corrente




EVIDENCE
Campo Descrizione
evi_id: int Id auto-incrementante
evi_etichetta: varchar Etichetta dell'evidence (ES: HDSEAGATE, MSDKINGSTON, ecc...)
evi_tipo: varchar Tipologia di evidence (Hard disk, Memory Card, ecc...)
evi_modello: varchar Model number dell'evidence
evi_seriale: tinyint Serial Number dell'evidence
evi_pwd: varchar Password dell'evidence
evi_pwd_used: tinyint 0 | 1. Se il campo è 1 allora la password è stata utilizzata per acquisire l'evidence.
evi_dimensione: varchar Dimensione dell'evicende (solo il numero)
evi_kbmbgbtb: varchar Dimensione dell'evidence (specificare se sono KB,MB,GB,TB)
evi_pathfoto: varchar Percorso della directory in cui si trovano le foto relative all'evidence corrente
evi_image1: varchar Nome della prima immagine (comprensivo di estensione) dell'evidence corrente
evi_image2: varchar Nome della seconda immagine (comprensivo di estensione) dell'evidence corrente
evi_image3: varchar Nome della terza immagine (comprensivo di estensione) dell'evidence corrente
evi_image_docx: varchar Nome della prima imagine scelta per la visualizzazione delle info utili ad essere incollate in consulenza (documento .docx)
ex_id_host: int Chiave esterna relativa all'Id dell'host a cui appartiene l'evidence corrente




HOST SPECIAL NOTE: rientrano negli host special tutti quei dispositivi che sarebbero degli evidence se appartenessero ad un Host normale. Siccome spesso vengono rinvenuti dispositivi (ES: hard disk, pen drive, ecc...) non appartenenti ad un PC, Notebook; allora rientrano tra gli host special
Campo Descrizione
ho_id: int Id auto-incrementante
ho_etichetta: varchar Etichetta dell'host (es. HDSEAGATE, PDKINGSTON, ecc...)
ho_seriale: varchar Serial number del dispositivo
ho_tipo: varchar Tipologia di Host (ES: Workstation, Notebook, ecc...)
ho_modello: varchar Model number del dispositivo
ho_dimensione: varchar Dimensione dell'host / dispositivo (solo il numero)
ho_kbmbgbtb: varchar Dimensione dell'host / dispositivo (specificare se sono KB,MB,GB o TB)
ho_pathfoto: varchar Percorso della directory in cui si trovano le foto relative all'host corrente
ho_image1: varchar Nome della prima immagine (comprensivo di estensione) dell'host corrente
ho_image2: varchar Nome della seconda immagine (comprensivo di estensione) dell'host corrente
ho_image3: varchar Nome della terza immagine (comprensivo di estensione) dell'host corrente
ho_image4: varchar Nome della quarta immagine (comprensivo di estensione) dell'host corrente
ho_image_docx: varchar Nome della prima imagine scelta per la visualizzazione delle info utili ad essere incollate in consulenza (documento .docx)
ho_image_docx2: varchar Nome della seconda imagine scelta per la visualizzazione delle info utili ad essere incollate in consulenza (documento .docx)
ex_id_caso: int Chiave esterna relativa all'Id del caso a cui appartiene l'host corrente
ex_id_indagato: int Chiave esterna relativa all'Id dell' indagato a cui appartiene l'host corrente




CLONE
Campo Descrizione
clo_id: int Id auto-incrementante
clo_tipoacq: varchar Tipologia di acquisizione
clo_altro_tipo: varchar Altra tipologia di acquisizione
clo_stracq: varchar Strumento di acquisizione
clo_md5: varchar MD5 calcolata dallo strumento di acquisizione
clo_sha1: varchar SHA1 calcolata dallo strumento di acquisizione
clo_sha256: varchar SHA256 calcolata dallo strumento di acquisizione
clo_log: varchar Nome del LOG (comprensivo di estensione)
ex_id_evi: int Chiave esterna relativa all'id dell'evidence a cui appartiene il log corrente
ex_id_host_special: int Chiave esterna relativa all'id dell'host special a cui appartiene il log corrente




LAVORAZIONE NOTE: questa tabella contiene le info relative allo stato di lavorazione di un determinato incarico e quale operatore se ne stà occupando
Campo Descrizione
id: int Id auto-incrementante
numero: varchar Numero del caso
pm: varchar PM responsabile dell'indagine
procura: varchar Procura che stà portando avanti l'indagine
dinizio: varchar Data di inizio dell'incarico conferito
gg: varchar Giorni permessi dal PM per portare a termine l'incarico
dfine: varchar Data di scadenza per la consegna della consulenza
ggrestanti: varchar Giorni restanti alla scadenza dell'incarico
copia: tinyint Flag: 0|1|NO. Se impostato a 1 la copia è stata eseguita. Se impostato a NO, non vi è necessità.
ftk: tinyint Flag: 0|1|NO. Se impostato a 1 è stato importato il caso in FTK. Se impostato a NO, non vi è necessità.
ief: tinyint Flag: 0|1|NO. Se impostato a 1 è stato analizzato con IEF(Internet Evidence Finder della Magnet Forensics). Se impostato a NO, non vi è necessità.
analisi: tinyint Flag: 0|1|NO. Se impostato a 1 è stata completata la fase di analisi. Se impostato a NO, non vi è necessità.
exprep: tinyint Flag: 0|1|NO. Se impostato a 1 è stata completata la fase di reportistica. Se impostato a NO, non vi è necessità.
dim: tinyint Flag: 0|1|NO. Se impostato a 1 è stata completata la documentazione tecnica dei dispositivi. Se impostato a NO, non vi è necessità.
allegati: tinyint Flag: 0|1|NO. Se impostato a 1 è stata completata la fase di imbustamento degli allegati con relative etichette incollate su di essi. Se impostato a NO, non vi è necessità.
liquidazione: tinyint Flag: 0|1|NO. Se impostato a 1 la consulenza è stata pagata dal cliente. Se impostato a NO, non vi è necessità.
difficolta: tinyint Flag: ROSSO|GIALLO|VERDE. ROSSO=Caso complesso/di dimensioni notevoli, GIALLO=media difficoltà/medie dimensioni, VERDE=difficoltà semplice/piccole dimensioni
progresso: tinyint Valore rappresentante la % di avanzamento del lavoro
note: varchar Eventuali note (ES: in attesa di consegna dei PIN/password, ecc...)
operatore: varchar Nome dell'operatore che si sta occupando del caso
last_upd_ggrestanti: date Data dell'ultimo aggiornamento dei giorni restanti alla scadenza dell'incarico




MAGAZZINO NOTE: questa tabella contiene le info relative al numero di "colli" da riconsegnare relativamente ad un determinato incarico/caso
Campo Descrizione
id: int Id auto-incrementante
procura: varchar Procura che stà portando avanti l'indagine
pm: varchar PM responsabile dell'indagine
dossier: varchar Numero del caso/dossier
plico: tinyint Numero di plichi
dataC: date Data di consegna
note: varchar Eventuali note




TIPO ACQUISIZIONE NOTE: questa tabella contiene le tipologie di acquisizione degli evidence
Campo Descrizione
acq_id: int Id auto-incrementante
acq_tipo: varchar Tipologia di acquisizione
acq_icon: varchar Nome dell'icona (comprensivo di estensione) rappresentante graficamente la tipologia di acquisizione




TIPO EVIDENCE NOTE: questa tabella contiene le tipologie degli evidence
Campo Descrizione
evi_id_tipo: int Id auto-incrementante
evi_tipo: varchar Tipologia di evidence
evi_default: tinyint 0|1. Se impostato a 1 non è eliminabile. Se impostato a 0 è eliminabile.
evi_icon: varchar Nome dell'icona (comprensivo di estensione) rappresentante graficamente la tipologia di evidence




TIPO HOST NOTE: questa tabella contiene le tipologie degli host
Campo Descrizione
ho_id_tipo: int Id auto-incrementante
ho_tipo: varchar Tipologia di host
ho_default: tinyint 0|1. Se impostato a 1 non è eliminabile. Se impostato a 0 è eliminabile.
ho_icon: varchar Nome dell'icona (comprensivo di estensione) rappresentante graficamente la tipologia di host




TIPO HOST SPECIAL NOTE: questa tabella contiene le tipologie degli host special
Campo Descrizione
hos_id_tipo: int Id auto-incrementante
hos_tipo: varchar Tipologia di host special
hos_default: tinyint 0|1. Se impostato a 1 non è eliminabile. Se impostato a 0 è eliminabile.
hos_icon: varchar Nome dell'icona (comprensivo di estensione) rappresentante graficamente la tipologia di host special




UTENTI NOTE: questa tabella contiene gli utenti di CFCM con relative credenziali di login
Campo Descrizione
ute_id: int Id auto-incrementante
ute_username: varchar Username
ute_password: varchar Password in formato MD5
ute_nome: varchar Nome dell'utente
ute_cognome: varchar Cognome dell'utente
ute_isadmin: int 0|1. Se impostato a 1 può visualizzare le impostazioni di amministrazione




TOOLS NOTE: questa è una tabella di appoggio temporaneo di informazioni utili ai tools di supporto al lavoro con FTK
Campo Descrizione
id: int Id auto-incrementante
item: int Item number degli elementi presenti nello strumento Ftk Toolkit
path: varchar Path degli elementi presenti in un dispositivo analizzato tramite Ftk toolkit
md5: varchar MD5 relativi agli elementi presenti in Ftk Toolkit
